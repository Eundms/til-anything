# AWS Lambda
> 서버리스

- 자동으로 확장 가능하며 가용성 높음, 코드에만 집중
- `런타임` 확인 필요 : `최대 지속 시간 15분`
- 완전관리형
- 이벤트 기반 프로세스에 적합 
  - 웹 사이트 요청 처리, 데이터 배치 처리, 경비 보고서 생성
  - 실시간 이미지 처리 : 이미지 크기 조정, 필터 적용, 최적화된 형식으로 스토리지에 저장
  - 뉴스 애그리게이터를 위한 개인화된 콘텐츠 전송 : 사용자와 사용작용할 때만 코드 실행하여 비용 절감
  - 온라인 게임의 실시간 이벤트 처리 : 사용량에 따라 비용의 규모가 조정, 피크 시간

# AWS 기반의 컨테이너 및 오케스트레이션 
> 컨테이너를 실제로 실행할 위치 : Amazon EC2 혹은 AWS Fargate

## Amazon Elastic Container Registry(Amazon ECR)
> 컨테이너 이미지를 저장하는 완전관리형 컨테이너 레지스트리, 버전관리 

- `OCI(Open Container Initiative) 표준`을 준수하는 컨테이너 이미지를 지원
- 표준 컨테이너 도구 및 명령줄 인터페이스(CLI)를 사용하여 Amazon ECR 리포지토리의 이미지를 푸시하고, 가져오고, 관리

## Amazon Elastic Container Service(Amazon ECS)
> EC2 인스턴스 유형이나 로드 밸런서 같은 애플리케이션의 컨테이너 이미지와 리소스를 정의해야 함

- Amazon EC2를 이용한 Amazon ECS : 인프라 완벽 제어, 중소기업 
- AWS Fargate를 이용한 Amazon ECS : 가변 트래픽, 스타트업 또는 소규모 팀


## Amazon Elastic Kubernetes Service(Amazon EKS) : 완전 관리형
> AWS에서 Kubernetes를 편리하게 실행

- Amazon EC2를 사용하는 Amazon EKS : 인프라를 완벽하게 제어해야 하는 기업, 대규모 워크로드
- AWS Fargate를 사용한 Amazon EKS : 서버를 관리할 필요가 없는 Kubernetes 유연성을 원하는 팀

## AWS Fargate 
> 완전관리형,Fargate는 컨테이너 호스팅 플랫폼, 서버리스 컴퓨팅 엔진

# AWS Elastic Beanstalk
> 웹 애플리케이션의 배포 및 스케일링,모니터링을 위한 관리형 서비스

- 애플리케이션 코드와 원하는 구성을 Elastic Beanstalk 서비스에 제공하면 환경 구축해줌 
- 환경 구성을 저장하고 다시 배포할 수 있음 

# AWS Batch
> 대규모 컴퓨팅 태스크를 관리, 수요에 따라 리소스 조정

- 서버 프로비저닝, 리소스 스케일링 또는 인프라 관리에 대해 걱정할 필요가 없음

# AWS Lightsail 
> 예측 가능한 월별 요금으로 가상 프라이빗 서버(VPS), 스토리지, 데이터베이스, 네트워킹을 제공하는 간소화된 클라우드 서비스

- 복잡한 인프라 없이도 웹 애플리케이션 설정 및 관리를 간소화
- 특정 유형의 애플리케이션 및 웹 사이트를 실행할 수 있는 쉽고 비용 효율적인 솔루션을 제공

# AWS Outposts
> AWS 서비스를 온프레미스 환경으로 확장하여 하이브리드 클라우드 아키텍처를 지원

-  AWS 서비스를 온프레미스 데이터 센터로 확장하여 양쪽 환경 모두에서 일관된 경험을 제공
