# 10. AWS 클라우드에서의 모니터링, 규정 준수, 거버넌스

# 리소스 모니터링 과정
1. 보안 시스템
- 리소스 보안 : 데이터, 시스템, 인프라에 무단 액세스, 사용, 공개, 중단, 수정 또는 파기가 일어나지 않도록 보호

2. 모니터링 활동
- 모니터링 : 시스템 활동, 네트워크 트래픽, 보안 이벤트를 지속적으로 관찰하고 분석하여 잠재적 위협이나 이상 징후를 탐지합니다.

3. 감사 실시
- 감사 : 보안 제어의 효과를 정기적으로 검토 및 평가하며, 모든 요구 사항을 충족하고 있고, 보안 정책 및 절차를 준수하는지 확인합니다.

4. 규정 준수 보장
- 규정 준수 : 조직의 보안 관행 및 제어가 관련 규정, 산업 표준, 계약상 의무에 따른 요구 사항을 충족하도록 보장합니다.

# 모니터링 도구 : AWS CloudWatch
## 구성 요소 
- 지표 : 리소스와 관련된 변수 
- 경보 : 지표의 임계값을 정의하고 알림을 보내거나 리소스를 자동으로 변경할 수 있음 
- 대시보드 : 한 곳에서 리소스를 모니터링하는데 사용할 수 있음
- 로그 : 모든 시스템, 애플리케이션 및 AWS 서비스의 로그를 중앙 집중화

## CloudWatch 로그 
  - AWS 리소스에서 로그 파일을 수집, 저장, 모니터링하여 로그 관리 및 분석을 중앙에서 실행
  - 로그를 보고, 검색하고, 필터링

## 장점
1. 다양한 소스의 지표를 중앙 엑세스 
- AWS 리소스, 온프레미스 서버 포함
- 평균 수리시간(MTTR)이 단축됨
- 총 소유 비용(TCO)등이 개선됨
2. 비즈니스에 집중 
3. 인사이트 도출하여 리소스 최적화 

# 감사 : AWS CloudTrail
> 모든 요청이 CloudTrail에 기록

## 구성 요소 
- 이벤트 : AWS계정 내에서 수행한 작업에 대한 세부 정보 캡처 
  - AWS 리전에서 발생한 지난 90일간의 관리 이벤트에 대해 조회/검색/다운로드 가능하며 변경 불가능함
  - 이벤트 기록을 보는 작업에는 CloudTrail 요금이 부과되지 않음
- 로그 : 이벤트를 모니터링하고 해당 이벤트를 로그 파일로 S3 버킷에 전송 
  - 지불 카드 산업(PCI) 및 의료보험의 양도 및 책임에 관한 법률(HIPPA) 규정 준수 증명 가능
- Insights :오류 발생률의 일반적인 패턴을 분석하고 다르면 이벤트 생성
  - 이상탐지 

# 규정준수 : AWS Artifact
-  AWS 보안 및 규정 준수 보고서와 일부 온라인 계약에 온디맨드로 액세스할 수 있는 무료 서비스
## AWS Artifact 계약
- 개별 계정 및 AWS Organizations 내 모든 계정에 대한 계약을 검토, 수락 및 관리
## AWS Artifact 보고서
- 서드 파티 감사 기관이 작성한 규정 준수 보고서를 제공


# AWS Config, AWS Audit Manager
## AWS Config 
- AWS 리소스 구성을 모니터링 및 기록하며 여러분이 구현하려는 구성과 비교하여 평가하는 서비스
- 내부 정책 및 규정을 준수할 수 있도록 리소스를 평가, 감사, 평가하는 작업을 지원

## AWS Audit Manager
- `증거 수집을 자동화`하여 감사 활동에 자주 요구되는 팀 간의 협업 작업을 줄일 수 있음
- 정책, 절차, 활동, 이른바 제어 활동이 효과적으로 운영되고 있는지 평가할 수 있음


# AWS Organizations
- 관리 계정 : 제어와 거버넌스 담당
- 조직 단위(OU) :  AWS Organization의 계정을 논리적으로 그룹화, 멤버 계정 또는 중첩된 OU 포함 가능
- 서비스 제어 정책 (SCP) : SCP는 각 계정의 사용자 및 역할이 액세스할 수 있는 AWS 서비스, 리소스, 개별 API 작업을 제한할 수 있는 정책. OU 또는 개별 멤버 계정에 적용 가능
- OU에 없는 멤버 계정 : 조직 단위의 요구 사항과 겹치지 않는 고유한 요구 사항이 있는 멤버 계정이 있을 경우, 해당 계정을 조직에 추가할 수 있습니다. OU 아래에 배치할 필요는 없습니다. 이 계정은 통합 결제 같은 이점을 계속 이용할 수 있습니다.

1. 조직 내의 여러 AWS 계정을 통합하고 중앙에서 관리할 수 있는 계정 관리 서비스

- 기본 계정 또는 상위 계정을 지정하면 다른 모든 계정은 기본 계정의 보조 계정 또는 하위 계정이 됨
- 결제는 기본 계정에 통합됨. 하위 계정에 대한 모든 결제는 상위 계정으로 이월됨. 최상위 수준에서 할인이 적용됨

2. 보안, 규정 준수 또는 예산 요구 사항을 충족하기 위한 계층적 계정 그룹화
- `조직 단위(OU)`로 그룹화 : `특정 규제 요구 사항을 충족하는 서비스에만 액세스`해야 하는 계정이 있을 경우 해당 계정을 단일 OU에 넣을 수 있음
- 개발자 OU에 속하는 계정이 있는 경우 : 조직의 기본 계정 관리자는 각 계정이 액세스할 수 있는 AWS 서비스 및 API 작업을 제어할 수도 있음
- `서비스 제어 정책(SCP)`을 사용하여 `조직의 멤버 계정에 대한 최대 권한을 지정`할 수도 있음 
  - 각 멤버 계정의 사용자 및 역할이 액세스할 수 있는 AWS 서비스, 리소스, 개별 API 작업을 제한할 수 있음

# 거버넌스 : AWS Control Tower
- AWS 클라우드의 모든 조직 및 계정에서 보안, 운영, 규정 준수에 대한 거버넌스 규칙을 대규모로 적용하고 관리할 수 있는 서비스

## 구성요소 
1. 대시보드
- 프로비저닝된 계정을 전사적으로 확인할 수 있도록 지속적인 감독을 제공
- 정책 시행을 위한 제어 기능을 갖추고 있으며, 규정을 준수하지 않는 리소스를 탐지하는 데 도움이 될 수 있음

2. Factory
- 새로운 계정의 프로비저닝을 표준화하는 구성 가능한 계정 템플릿

3. 제어
- 거버넌스를 제공하는 상위 수준 규칙

4. 랜딩 존
- 보안 및 규정 준수 모범 사례를 기반으로 잘 설계된 다중 계정 환경
- 규제하고자 하는 모든 조직 단위(OU), 계정, 사용자, 리소스를 보존하는 전사적인 컨테이너

## AWS Service Catalog
-  AWS 리소스의 카탈로그를 생성 및 공유하고 구성
- 기준 네트워킹 리소스 및 보안 도구를 배포하여 일관성 있게 관리

## AWS License Manager
- 소프트웨어 라이선스를 관리하고 라이선스 비용을 미세 조정할 수 있는 서비스


# AWS Health 
- AWS 클라우드 리소스의 상태에 영향을 미치는 이벤트 및 변경 사항을 위한 유용한 데이터 소스
- 계정별 상태 정보를 확인하고, AWS Health 이벤트 업데이트를 받을 수 있음


# AWS Trusted Advisor
