# 저장된 데이터 및 전송 중인 데이터를 보호하기 
## 저장된 데이터 암호화 
### Amazon S3
기본적으로 모든 새 버킷에 암호화가 구성되어 있습니다. 
또한 S3 버킷에 업로드되는 모든 새 객체는 저장 시 자동으로 암호화됩니다.

### Amazon EBS 볼륨 및 스냅샷
저장 시 암호화할 수 있습니다. EC2 인스턴스의 부팅 볼륨과 데이터 볼륨도 모두 암호화할 수 있습니다.


### AWS Key Management Service(KMS)
AWS KMS를 사용하여 암호화 키를 생성하고 관리할 수 있습니다. 그런 다음, 이러한 키를 사용하여 데이터를 암호화하고 해독할 수 있습니다. 또한 광범위한 서비스 및 애플리케이션에서 키 사용을 제어할 수 있습니다. 예를 들어 키를 관리할 수 있는 IAM 사용자 및 역할을 지정할 수 있습니다. 키는 AWS KMS를 벗어나지 않으며, 키를 일시적으로 비활성화하여 더 이상 사용하지 못하도록 설정할 수 있습니다.

### Amazon Macie
저장된 민감한 데이터를 모니터링하여 해당 데이터가 안전한지 확인할 수 있습니다. Macie는 기계 학습(ML) 및 자동화를 사용하여 Amazon S3에 저장된 민감한 데이터를 검색합니다. Macie를 사용하면 보안 태세를 평가할 수 있는데, 이는 규정 준수 요구 사항을 충족하는 데 특히 유용합니다.


## 전송 중 데이터 암호화 

인증서를 중앙에서 관리하는 AWS 서비스를 `AWS Certificate Manager(ACM)`라고 합니다. 
ACM의 장점은 연결된 온프레미스 리소스 외에도 다양한 AWS 서비스를 보호하는 데 사용할 수 있다는 겁니다.
