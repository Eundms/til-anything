# Amazon Elastic Block Store(EBS) : 블록 스토리지
-  물리적 하드 드라이브처럼 EC2 인스턴스에 연결되는 `영구적`이고 지연 시간이 짧은 블록 수준 스토리지 볼륨을 제공
- 스냅샷을 사용하여 가용 영역 간에 EBS 볼륨을 쉽게 마이그레이션 가능
- 인스턴스 유형 변경 가능
- 재해 복구 가능
- 비용 최적화 가능
- 성능 튜닝 가능

## vs EC2 인스턴스 스토어
  - 중지되거나 종료되면 `데이터가 손실`되므로 
  - 버퍼, 캐시, 스크래치 데이터 같은 임시 메모리 기반의 스토리지 요구 사항에 가장 적합

## EBS 스냅샷
- 데이터 보호 및 복구 : 특정 시점 백업을 사용하여 손상, 실수로 인한 삭제 또는 시스템 실패 시 데이터를 빠르게 복구
- 운영 유연성 : 스냅샷을 사용하면 리전 간 데이터 마이그레이션, 볼륨 크기 조정, 볼륨 복제, AWS 계정 전체의 데이터 공유 같은 작업을 수행할 수 있습니다.
- 비용 효율성 : 증분 백업 기술을 사용하여, 초기 백업 후 변경된 블록만 저장하므로 스토리지 비용과 백업 시간이 줄어듭니다.

## Amazon Data Lifecycle Manager
- 알정에 따라 EBS 스냅샷 생성, 보존, 삭제 작업을 자동화


# Amazon Simple Storage Service(S3) : 객체 스토리지
- 데이터를 플랫 주소 공간의 객체로 관리하는 데이터 스토리지 아키텍처
- 데이터 자체, 메타데이터, 고유 식별자 또는 키
  - 각 Amazon S3 객체는 키를 기준으로 버킷 내에서 고유하게 식별
- 사실상 무제한 스토리지
- 객체 수명 주기 관리 가능
- 기본적으로 비공개 
    - Amazon S3 버킷 정책은 S3 버킷에만 연결할 수 있는 리소스 기반 정책입니다. S3 버킷 정책은 해당 버킷에 있는 모든 객체 외에도, 버킷에서 허용되거나 거부되는 작업을 지정합니다.
    - 사용자, 그룹 또는 역할이 S3 리소스에서 어떤 작업을 수행할 수 있는지 제어하는 권한은 자격 증명 기반 정책을 사용하여 구성됩니다. 이러한 정책은 S3 리소스 자체가 아닌 자격 증명에 직접 연결됩니다. 이러한 정책을 사용하여 사용자가 어떤 S3 버킷 및 객체에 연결할 수 있고, 어떤 작업을 수행할 수 있는지 지정할 수 있습니다.
    - Amazon S3는 저장 중인 데이터와 전송 중인 데이터를 모두 보호하는 암호화 기능을 제공합니다. 이러한 암호화 기능은 데이터 기밀을 유지하고, 다양한 보안 표준 및 규정을 준수하는 데 도움을 줍니다. 이러한 기능은 다음과 같습니다.


S3 Block Public Access 설정은 버킷 정책을 재정의하므로, 정책에서 허용하는 경우에도 퍼블릭 액세스를 차단합니다.




- 광범위한 사용 사례
  - 콘텐츠 배포, 정적 웹 사이트 호스팅, 미디어 파일 전송
  - 애플리케이션 데이터 스토리지, 아카이빙, 데이터 레이크, 규정 준수 기반의 데이터 보존 같은 용도


# Amazon Elastic File System(Amazon EFS), Amazon FSx: 파일 스토리지
- 네트워크를 통해 액세스할 수 있는 공유 파일 시스템을 제공
## Amazon Elastic File System(Amazon EFS)
- AWS 클라우드 서비스와 온프레미스 리소스에서 사용할 수 있는 확장 가능한 완전관리형 NFS 파일 시스템
## Amazon FSx
- Windows, Lustre, NetApp ONTAP 같은 널리 사용되는 파일 시스템을 위한 완전관리형 파일 스토리지 서비스

# 추가 스토리지 서비스
## AWS Storage Gateway
- 온프레미스에서 사실상 무제한 클라우드 스토리지에 액세스할 수 있는 완전관리형 하이브리드 클라우드 스토리지 서비스입니다.

## AWS Elastic Disaster Recovery
- 물리적 서버, 가상 서버, 클라우드 기반 서버를 AWS로 복구하는 과정을 간소화하는 완전관리형 서비스입니다.
