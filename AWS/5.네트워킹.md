# VPC
- AWS 클라우드의 `논리적으로 격리`된 섹션
- 네트워크 리소스와 보안을 제어하는 데 도움이 됨

## 서브넷 : VPC를 관리 가능한 작은 섹션으로 나눈 것
- 리소스를 구성하는 데 사용되며 퍼블릭 또는 프라이빗 방식으로 액세스할 수 있음

## 인터넷 게이트웨이
- 퍼블릭 서브넷이 인터넷 연결을 위해 연결되어 있음 

## 가상 프라이빗 게이트웨이 
- 가상 프라이빗 게이트웨이를 사용하여 승인된 네트워크에서 나오는 트래픽만 VPC로 들어가도록 허용할 수 있음
- 이를 사용하여, `VPC`와 `프라이빗 네트워크(예: 온프레미스 데이터 센터 또는 회사 내부 네트워크)` 간에 `VPN 연결`을 설정할 수 있음
  - 보안 연결을 사용하여 인터넷을 통해 회사 데이터 센터를 연결 (단, 리소스를 일반 대중과 격리해야 함)

# AWS 클라우드에 연결할 수 있는 추가적인 방법
## AWS Client VPN : 원격 작업자를 AWS 클라우드 리소스에 안전하게 연결 
- `원격 작업자`와 `온프레미스 네트워크`를 클라우드에 연결하는 데 사용할 수 있는 네트워킹 서비스
- 탄력적인 완전관리형 VPN 서비스

## AWS Site-to-Site VPN : 사이트를 다른 사이트에 안전하게 연결
- 데이터 센터 또는 지사 같은 `온프레미스 네트워크`와 `Amazon VPC의 리소스` 간에 암호화된 보안 연결을 설정
- 원격 위치 간의 애플리케이션 마이그레이션 및 보안 통신에 사용

## AWS Private Link : 다른 VPC에서도 리소스를 안전하게 연결
>  직접 보유한 VPC의 리소스에 연결하듯이, VPC를 다른 클라우드 제공업체의 리소스에 비공개로 연결할 수 있음

- VPC를 서비스 및 리소스에 비공개로 연결하는 데 사용할 수 있는 가용성과 확장성이 뛰어난 기술

## AWS Direct Connect : 대역폭 증가를 위한 전용 프라이빗 연결 
- 네트워크와 VPC 간에 전용 프라이빗 연결을 설정
- 네트워크 비용을 줄이고 대역폭의 양을 늘림
- 마이그레이션을 위한 대역폭 요구사항 제공 
  - 한 회사에서 데이터 웨어하우스 및 데이터 백업을 사용하여 온프레미스 데이터 센터의 대규모 마이그레이션을 수행하고 있습니다. 이 회사에는 마이그레이션 동안 많은 대역폭 요구 사항을 충족할 수 있는 솔루션이 필요합니다. 온프레미스 데이터 센터의 일부를 하이브리드 클라우드 솔루션용으로 유지할 예정이므로, 이 솔루션은 이동 후에도 지속적인 데이터 전송에 사용하고자 합니다.

## AWS Transit Gateway 
- 중앙 허브를 통해 VPC와 온프레미스 네트워크를 연결
- 리전 간 피어링은 AWS Global Infrastructure를 사용하여 전송 게이트웨이를 서로 연결

## Network Address Translation(NAT) 게이트웨이
- 프라이빗 서브넷의 인스턴스에서 VPC 외부 서비스에 연결할 수 있지만, 외부 서비스에서는 해당 인스턴스와의 연결을 시작할 수 없음 

## Amazon API Gateway
- API를 생성, 게시, 유지 관리, 모니터링, 보호하는 AWS 서비스
