
# AWS 클라우드의 주요 이점
> 고가용성, 내결함성 

1.	고정 -> 가변 
2.	규모의 경제
3.	용량 추정 불필요
    - 동적 확장
4.	속도 및 민첩성 향상
    - 변화하는 신속 대응 가능
5.	데이터센터 운영 및 유지 관리에 비용 투자 불필요
6.	몇분만에 전 세계에 배포 

# AWS 글로벌 인프라 
- 한 리전에 3개이상의 AZ(가용구역)가 있고, 각 AZ 내에는 중복 전원, 네트워킹, 연결성을 갖춘 하나 이상의 개별 데이터 센터가 있다 
- 멀어질 수록 지연 시간이 길어지므로 가까운 AWS 리전 선택 

# AWS 공동 책임 모델 
- AWS : 하드웨어, AWS 글로벌 인프라, 물리적 네트워크, 하이퍼바이저 계층
- 고객 : 운영체제(여기 접근하는 암호화키)에 패치 적용 책임, 애플리케이션, 클라이언트 측 데이터 암호화

## 서비스마다 다름
### 완전관리형 서비스 (Lambda)
> 코드 작성 및 배포에만 집중

  - 고객:  클라이언트 측 데이터 암호화, 고객 데이터
  - AWS : `서버측 암호화`/ `네트워크 트래픽 보호`/ `플랫폼 및 애플리케이션 관리` / `OS, 네트워크, 방화벽 구성` / 컴퓨팅, 스토리지, 데이터베이스, 네트워킹용 소프트웨어 / 하드웨어 AWS 글로벌 인프라 

### 관리형 서비스 (ELB, SNS, SQS)
> 배포 옵션 선택 및 환경 설정 구성

  - 고객: 네트워크 트래픽 보호, 서버측 암호화, 클라이언트 측 데이터 암호화, 고객 데이터
  - AWS : `플랫폼 및 애플리케이션 관리` / `OS, 네트워크, 방화벽 구성` / 컴퓨팅, 스토리지, 데이터베이스, 네트워킹용 소프트웨어 / 하드웨어 AWS 글로벌 인프라 

### 비관리형 서비스(ex - EC2) 사용하는 경우 
> 운영체제, 보안패치, 네트워크 설정 구성

  - 고객 : `OS, 네트워크, 방화벽 구성` / `플랫폼 및 애플리케이션 관리` /  `네트워크 트래픽 보호` / `서버측 암호화` / 클라이언트 측 데이터 암호화, 고객 데이터
  - AWS : 컴퓨팅, 스토리지, 데이터베이스, 네트워킹용 소프트웨어 / 하드웨어 AWS 글로벌 인프라
