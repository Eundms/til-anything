# 클라우드 기반 컴퓨팅

## EC2 특징
 - 유연성
 - 비용 효율성
 - 확장성

## 멀티테넌시 
  - 기본 물리적 호스트 컴퓨터를 다른 여러 인스턴스와 공유 
  - VM이 서로 격리된 상태에서도 호스틑에서 제공하는 리소스를 계속 공유하도록 보장 : 하이퍼바이저
  - AWS : 기본 호스트, 하이퍼바이저, 인스턴스 간의 격리 관리 

## 인스턴스 생성 및 사용 방법 
1. 인스턴스 시작
  - 운영체제 정의 : AMI선택(운영체제 + 내장된 애플리케이션, 사용자 지정도 가능)
  - 타입 : CPU, 메모리, 네트워크 성능 같은 기본 하드웨어 리소스 결정
  - 키페어
  - 네트워크 세팅
  - 스토리지 옵션 
2. 인스턴스에 연결 
  - SSH, RDP 프로토콜로 연결 
  - AWS System Manager : 엑세스 
3. 인스턴스 사용


> 종료 = 삭제, 중지 = 일시정지

## AMI
### AMI 사용 방법
- 필요에 맞게 맞춤 설정된 특정한 구성 및 소프트웨어로 사용자 지정 AMI를 구축하여 자체 AMI를 생성
- 일반 운영 체제 및 소프트웨어용으로 설정된 사전 구성된 AWS AMI를 사용
- 특정한 사용 사례에 맞게 설계된 특수 소프트웨어를 서드 파티 공급업체가 제공하는 AWS Marketplace에서 AMI를 구매
### AMI 반복성
- 모든 새 인스턴스에 대해 일관된 환경을 통해 반복성 제공
- 구성 동일, 배포 자동화, 규모 조정 시 유용, 오류 줄임, 대규모 환경 관리 간소화 

## 인스턴스 패밀리 
### 1. 범용 인스턴스 
- 컴퓨팅, 메모리, 네트워킹 리소스 균형 있게 제공 
- 웹서비스, 코드 리포지토리 등 워크로드가 어떻게 작동할지 모르는 경우

### 2. 컴퓨팅 최적화 (CPU)
- `계산`
- 게임 서버, 고성능 컴퓨팅, ML

### 3. 메모리 최적화
- 대규모 데이터 메모리에서 처리 
- 대규모 데이터세트, 신속한 쿼리 제공 

### 4. 가속 컴퓨팅 (GPU)
- GPU, `하드웨어 가속기`
- 부동 소수점 수 계산, 그래픽 처리 또는 데이터 패턴
- CPU에서 실행되는 소프트웨어보다 더 효율적으로 기능을 수행하는 보조 프로세서

### 5. 스토리지 최적화 
- 로컬에 `저장된 데이터`에 대해 고성능이 필요한 워크로드에 적합

## AWS 리소스 프로비저닝하는 방법
- AWS Management Console 
  - 웹 인터페이스 : 신속 액세스, 검색, 간소화된 워크플로
  - 모바일 앱 : 리소스 모니터링, 경보 확인, 청구서 확인, 여러 로그인한 ID 지원
- AWS Command Line Interface(AWS CLI)
  - 직접 관리, 자동화, 스크립팅
- AWS SDK
  - API 제공하여 애플리케이션에 AWS 서비스 통합할 수 있도록 함 

## EC2요금
### 온디맨드 인스턴스
- 워크로드 테스트(`시작`)
- 선불 지급 또는 장기 `약정 없이` 사용한 컴퓨팅 용량에 대해서만 비용을 지불합니다.

### 예약 인스턴스(RI)
- `안정적인 상태`의 워크로드, `꾸준한` 상태, 사용량 예측이 가능한 워크로드 
- `특정 인스턴스 패밀리 및 AWS 리전`을 사용하여 `예측 가능한 워크로드`에 대해 1년 또는 3년 기간으로 약정을 체결하면 최대 75%까지 비용을 절감할 수 있습니다.
  - 전액 선결제 : 약정 체결시 전액 지불
  - 부분 선결제 : 약정 체결시 금액 일부 지불

### 절감형 플랜
- 일정한 `사용량`에 약정 체결
    - 약 요금은 사용 여부와 관계없이 온디맨드 요금으로 청구되며, 실행한 인스턴스에 대해서만 비용을 지불하면 됩니다. 이는 현재 또는 미래의 비즈니스 크리티컬 워크로드에 대한 용량 요구 사항이 엄격한 경우에 적합합니다.
- 1년 또는 3년 동안 일관된 사용 수준을 유지하기로 약정하여 다양한 인스턴스 유형 및 서비스에서 최대 72%의 비용을 절감할 수 있습니다.

### 스팟 인스턴스
- 중요성이 낮으면서 `내결함성`이 있는 작업에 사용, `시간에 민감하지 않은` 작업
- 온디맨드 가격에서 최대 90% 할인된 가격으로 `예비 컴퓨팅 용량`에 입찰하며, AWS에서 인스턴스를 회수할 때 `중단할 수 있는` 유연성이 제공됩니다.
  - 2분동안 경고 -> 꺼짐 

### 전용 호스트
- 는 Windows 또는 SQL Server 같은 보안에 민감한 워크로드나 `라이선스` 관련 워크로드
- `전체 물리적 서버를 독점적으로 사용`할 수 있도록 예약합니다. 이 옵션은 완전한 제어 기능을 제공하며 엄격한 보안 또는 라이선싱 요구 사항이 있는 워크로드에 적합합니다.
    - 인스턴스 배치 및 리소스 할당을 완벽하게 제어할 수 있음

### 전용 인스턴스
- `사용자의 계정 전용 하드웨어에서 실행되는 인스턴스`에 대해서만 비용을 지불합니다. 이 옵션을 사용하면 `다른 AWS 고객과 격리`할 수 있습니다.
- 전용 인스턴스는 실행할 물리적 서버를 선택하지 않아도 격리 기능을 제공

# 자동 스케일링 및 로드 밸런싱
## Amazon EC2 스케일링
### 확장성 
  - 리소스를 추가하여 `증가한 로드를 처리`할 수 있는 시스템 능력
  - 장기적인 용량 계획에 중점 
### 탄력성
   - `실시간` 수요에 따라 자동으로 스케일 업하거나 다운 
   - `비용 효율성` 및 최적의 리소스 사용 제공

## Amazon EC2 Auto Scaling (= Scaling Out)
>  EC2 인스턴스 수를 자동으로 조정
> Amazon CloudWatch 서비스를 사용하여 지표를 수집하고 모니터링할 수 있음 

### 개념 : 스케일링 아웃, 업 
- 아웃 : 병렬 수행
- 업 : 병렬, 성능 강화 , 항상 필요한 건 아님

### 방식 : 동적 스케일링, 예측 스케일링
- 동적 : 수요 변화에 따라 `실시간 조정`
- 예측 : `예상 수요`에 따라 적절한 수의 인스턴스를 선제적으로 예약 

### Auto Scaling Group 설정 가능
- 최소 용량 : 최소 인스턴스의 수 
- 원하는 용량 : 이상적인 인스턴스 수 (기본 : 최소)
- 최대 용량 : 상한선을 설정하여 초과 스케일링 방지, 비용 제어

## Elastic Load Balancing(ELB) 사용하여 트래픽 전달
> Auto Scaling과 같이 동작, 다만 책임은 트래픽 분산

- 탄력적이라는 말은 시간당 비용 추가 없이 트래픽에 따라 스케일 업하거나 스케일 다운할 수 있는 기능
- 백엔드 인스턴스와 프론트엔드 인스턴스의 연결을 관리
- 리전 서비스이므로, ELB는 각 프론트엔드 인스턴스가 백엔드 인스턴스로 전달하는 데 사용하는 단일 URL

### 장점
1. 효율적인 트래픽 분배
  - 균일 분산

2. 오토스케일링 
- 새 인스턴스가 가동되고, ELB에 트래픽을 처리할 준비가 완료되었음을 알림 

3. 간소화된 관리 
- 티어 간 결합 해제, 유지 관리, 업데이트, 장애 조치 처리 

### 라우팅 방법 
1. 라운드 로빈 : 주기적 균등 분배
2. 최소 연결 : 활성화된 연결 수가 가장 적은 서버로 트래픽 라우팅
3. IP 해시 : 트래픽을 동일한 서버로 일관되게 라우팅
4. 최소 응답 시간 : 응답 시간이 가장 빠른 서버로 트래픽을 전달하여 지연 시간 최소화

## 메시징 및 대기열 처리 
### Amazon SQS
  - 메시지 대기  
  - 서비스 분리, 데이터 손실 방지
  - 소프트웨어 구성 요소 간에 모든 볼륨의 메시지를 전송, 저장, 수신 가능 

### SNS 
- 게시, 구독 서비스 
- 메시지 전송
