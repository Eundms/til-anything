# 리전 또는 여러 시전을 선택하는 방법
1. 특정한 데이터 규정 준수 규제해야 하는 경우
- 일반 데이터 보호 규정(GDPR)
2. 지연 시간 감소를 위한 근접성
3. 기능 가용성 : 필요한 기능이 있는 AWS 리전 사용 
4. 요금 : 현지 조세 체계 및 에너지 비용에 따라 차이남

# 가용성이 높은 시스템을 만드는 방법
1. 여러 가용 영역(AZ)을 사용 : 더 빠른 재해 복구, 비즈니스 연속성 개선, 지연 시간 감소, 규정 준수를 지원
  - 다른 영역의 실패와 격리하도록 설계된 1개 이상의 데이터 센터를 갖춘 별도의 개별 위치
  - usa-east-a, usa-east-b
2. 여러 AWS 리전에 애플리케이션을 배포
  - 세계의 물리적 위치 
  - usa-east


# AWS 엣지 로케이션 
- 리전과 분리되어 있음
- 표준 리전 밖에 있는 사용자에게 더 빠른 콘텐츠 액세스를 제공하는 글로벌 엣지 네트워크

## CloudFront
- 지연 시간이 짧아야 하는 콘텐츠에 사용자가 액세스할 수 있도록 이미지, 동영상 및 기타 리소스 등의 항목을 캐시
## Route 53
- 최종 사용자를 인터넷 애플리케이션으로 라우팅하는 도메인 이름 시스템(DNS) 
## Aws Global Accelerator
## AWS Outposts 
- 온프레미스에서 AWS 서비스를 실행

# 코드형 인프라(IaC) 및 CloudFormation 
- 클라우드 리소스 배포를 자동화 CloudFormation : IaC사용 
- 여러 계정 또는 여러 리전에 동일한 템플릿을 배포
